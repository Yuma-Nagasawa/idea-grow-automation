MDEフル探索：指数関数的に増える生産性構造
0. 定義

ここでの生産性をこう定義します。

生産性 = 1つの入力・種・知識・行動から、どれだけ多くの有用な成果物を生み出せるか

単なる作業量ではなく、

1 → N の変換効率

です。

1. Reality Structuring

普通の生産性は線形です。

1時間働く → 1成果物
10時間働く → 10成果物

しかし指数関数型は違います。

1成果物 → 次の成果物を生む仕組みになる
仕組み → さらに仕組みを生む
知識 → 新しい知識生成能力を増やす

つまり、成果物が「消費されるもの」ではなく、再生産する資産になります。

2. 中核仮説

指数関数的な生産性の本質はこれです。

成果物が、次回以降の成果物生成コストを下げるか、生成可能範囲を広げるとき、生産性は複利化する。

例：

記事を書く
↓
記事テンプレートができる
↓
次の記事が早くなる
↓
読者の反応データが貯まる
↓
より刺さる記事が書ける
↓
教材化できる
↓
講座・本・AIプロンプト・営業資料に転用できる

このとき、1つの記事は単なる記事ではなく、生産性を上げる母体になります。

4. 指数関数化の条件

生産性が複利化するには、最低でも次の4条件が必要です。

1. 成果物が再利用可能である

一回限りの作業では伸びにくいです。

単発資料 → 弱い
資料テンプレート → 強い
資料生成プロンプト → さらに強い
資料生成システム → かなり強い
2. 成果物が次の入力になる
記事 → 講座
講座 → 顧客理解
顧客理解 → 新講座
新講座 → ブランド
ブランド → 集客

このように、出力が次の入力になると回転します。

3. 学習が蓄積される

やりっぱなしでは複利になりません。

必要なのは、

何が効いたか
何が効かなかったか
どこで詰まったか
次はどう直すか

を残すことです。

4. 配布・複製コストが低い

指数関数化しやすいのは、デジタル資産です。

note
Udemy
YouTube
プロンプト
マクロ
テンプレート
チェックリスト
ナレッジカード
AIエージェント

複製コストが低いほど、1つの成果物が広がりやすいです。

5. 逆に、指数関数化しない構造

次のようなものは伸びません。

毎回ゼロから作る
成果物を保存しない
改善ログを残さない
他人に渡せない
再利用できない
自分しか使えない
検証しない

これは労働集約型です。

頑張っても、

1 → 1

のままです。

6. 生産性の成長モデル

かなり単純化すると、こうです。

次回の生産力
= 現在の生産力
+ 今回作った再利用資産
+ 今回得た学習
+ 今回増えた配布経路

よりMDE風に書くと、

P(t+1) = P(t) × (1 + A + L + D - F)
P = 生産性
A = 資産化率
L = 学習率
D = 配布・転用力
F = 摩擦・複雑化・疲労・管理コスト

指数関数化するには、

A + L + D > F

である必要があります。

9. MDE的結論

指数関数的な生産性とは、

たくさん作業する能力ではなく、作ったものが次の生産能力を増やす構造を設計する能力

です。

より短く言うと、

生産性の本質は、成果物を消費物ではなく、自己増殖する知識資産に変換すること。

です。
